# Apple Developer 등록 고객 안내 (Client FAQ — Apple Developer Enrollment)

**Korean primary. Send this to the client before submitting Apple Developer enrollment.**

---

## 안녕하세요 — 이 안내를 드리는 이유

귀사의 Apple Developer 계정 개설 대행을 담당하게 되었습니다. 전체 절차는 주식회사 포텐셜이 진행하지만, Apple이 **귀사에 직접 연락하는 순간이 한 번** 있습니다. 그 순간의 대응 방법만 안내드립니다.

## 귀사가 해야 할 일 — 약 1분

Apple의 심사팀이 **제출 후 1~3 영업일 이내에 귀사의 대표번호로 SMS 문자를 보냅니다**. (드물게 전화를 거는 경우도 있습니다.)

### 1. SMS를 받으시면

- 발신자: Apple 또는 `+1-408-` 로 시작하는 미국 번호
- 내용 예시: *"Your Apple Developer enrollment verification code is: 123456"*

### 2. 받으신 SMS를 그대로 포텐셜 담당자에게 전달

KakaoTalk 1:1 채팅방에 복사·붙여넣기 해주시면 됩니다. **10분 내에 전달해 주시기 바랍니다** (코드가 만료됩니다).

```
[고객이 KakaoTalk에 붙여넣는 예시]
Apple: Your Apple Developer enrollment verification code is: 123456
```

포텐셜 담당자가 해당 코드를 포털에 입력하여 인증을 완료합니다.

## 전화가 오는 경우 (드묾)

간혹 Apple 심사원이 영어로 전화를 걸어옵니다. 약 60초 통화이며, 질문은 2가지 뿐입니다:

**질문 1**: *"Did you authorize Potential Inc. to enroll \<귀사 영문명\> in the Apple Developer Program?"*
→ **답변**: *"Yes."*

**질문 2**: *"Are you the CEO / authorized signatory of \<귀사 영문명\>?"*
→ **답변**: *"Yes."*

통화 끝. 전화 거절하시지 마시고 받아 주세요 — 거절하면 Apple이 심사를 반려합니다.

## 예상 일정

| 시점 | 이벤트 | 귀사 대응 |
|---|---|---|
| D-day | 포텐셜이 Apple에 등록 신청 제출 | 없음 |
| D+1 ~ D+3 | Apple SMS 또는 전화 수신 | SMS 포워딩 (30초) 또는 전화 응답 (60초) |
| D+1 ~ D+5 | Apple 최종 승인 | 없음 |
| D+5 ~ D+7 | 포텐셜이 Developer 계정 최종 셋업 + 전달 | 없음 |

## 안 받으면 어떻게 되나요?

- SMS 안 보시고 10분 경과 → 코드 만료 → 포텐셜이 재전송 요청 → 다시 SMS 도착
- 전화 안 받고 3일 경과 → Apple이 심사 반려 → 처음부터 재신청

불편하시겠지만, **등록 기간 2~3일 동안만** SMS와 전화 주시해 주시면 됩니다.

## 자주 묻는 질문

**Q. 포텐셜 쪽 번호로 SMS 오게 할 수 없나요?**
A. Apple은 대표이사 번호 인증을 요구합니다. 귀사 번호에서 포텐셜 번호로 포워딩 설정은 가능하지만, 월 사용료가 발생합니다 (070 VoIP, 약 5,000원/월). 단기 1회성 등록이라면 KakaoTalk 포워딩이 가장 효율적입니다.

**Q. 신용카드는 왜 필요한가요?**
A. Apple Developer Program은 연 $99 (약 13만 원)의 연회비가 있습니다. 귀사 명의의 법인카드로 결제합니다. 포텐셜이 카드 정보를 대신 소지하지 않습니다 — 1Password 공용 보관소에만 저장됩니다.

**Q. 계정은 누구 소유인가요?**
A. 법적 소유자는 귀사입니다 (대표이사). 포텐셜은 "Admin" 권한을 부여받아 실무 운영을 수행합니다. 언제든지 포텐셜의 Admin 권한을 해지하실 수 있으며, 해지 시 즉시 반납됩니다.

**Q. 2FA (이중 인증)은 어떻게 되나요?**
A. Apple ID 2FA는 귀사 대표이사의 휴대폰에 설정되지 않고, 1Password 내 OTP 필드로 설정됩니다. 포텐셜과 귀사가 같은 1Password 보관소를 공유하므로 양쪽 모두 OTP 코드를 실시간 확인할 수 있습니다. 포텐셜이 운영을 중단하더라도 귀사는 계속 접근 가능합니다.

**Q. 등록 실패 시 비용은?**
A. Apple 심사 반려 시 포텐셜 대행 수수료는 환불되지 않지만, 재제출은 무상으로 진행합니다. Apple의 $99 연회비는 승인 후에만 청구되므로 반려 시 손실 없습니다.
