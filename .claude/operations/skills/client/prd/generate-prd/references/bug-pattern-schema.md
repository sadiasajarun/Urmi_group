# 버그 패턴 분류 스키마

이 파일은 `bug-patterns.md`의 구조와 각 패턴의 필수 필드를 정의한다.

---

## 파일 구조

```markdown
# Bug Pattern Knowledge Base
> Last updated: {YYYY-MM-DD}
> Total patterns: {N}
> Sources: {N} projects, {M} bug reports

## Category: {카테고리명}

### Pattern: {패턴명}
- **빈도**: {N}회 ({높음/중간/낮음})
- **증상**: {사용자가 경험하는 증상}
- **근본 원인**: {기술적 원인}
- **영향 화면**: {영향받는 화면/기능 목록}
- **예방 스펙**: "{PRD에 삽입할 구체적 스펙 문장}"
- **사례**: [{ProjectName #IssueNumber}, ...]
```

---

## 패턴 필수 필드

| 필드 | 필수 | 설명 |
|------|:----:|------|
| **빈도** | ✅ | 발견 횟수 + 등급 (1-2회: 낮음, 3-4회: 중간, 5회+: 높음) |
| **증상** | ✅ | 사용자 관점에서 경험하는 문제 (기술 용어 최소화) |
| **근본 원인** | ✅ | 왜 발생하는지 (기술적 원인) |
| **영향 화면** | ✅ | 이 패턴이 적용될 수 있는 화면/기능 유형 |
| **예방 스펙** | ✅ | PRD에 직접 삽입 가능한 구체적 스펙 문장 (따옴표로 감싸기) |
| **사례** | ✅ | 출처 (프로젝트명 + 이슈번호 or 파일명) |

## 카테고리 분류

### 표준 카테고리

| 카테고리 | 포함 범위 |
|---------|----------|
| **인증/로그인** | 로그인, 회원가입, 소셜 로그인, 토큰, 세션, 비밀번호 |
| **이미지/파일 업로드** | 파일 크기, 포맷, 업로드 실패, 이미지 리사이징 |
| **리스트/페이지네이션** | 무한 스크롤, 페이지 번호, 정렬, 필터, 빈 상태 |
| **푸시 알림** | 알림 수신, 권한, 딥링크, 포그라운드/백그라운드 |
| **결제/인앱구매** | 결제 플로우, 환불, 영수증, 동시 결제 |
| **실시간 데이터** | 채팅, 웹소켓, 실시간 업데이트, 동기화 |
| **관리자 대시보드** | 통계, 필터, 내보내기, 벌크 액션, 권한 |
| **검색/필터** | 검색 결과, 필터 조합, 자동완성, 최근 검색 |
| **네비게이션/라우팅** | 딥링크, 뒤로가기, 탭 전환, 히스토리 |
| **데이터 동기화** | 오프라인, 충돌 해결, 캐시, 낙관적 업데이트 |
| **폼/입력** | 유효성 검사, 자동저장, 다단계 폼, 파일 첨부 |
| **성능** | 느린 로딩, 메모리 누수, 렌더링 지연 |

### 카테고리 확장 규칙

- 위 표준 카테고리에 맞지 않는 버그 → 새 카테고리 생성 가능
- 카테고리명은 **기능 영역** 기준 (기술 스택 기준 아님)
- 예: "React 렌더링" (❌) → "리스트/페이지네이션" (✅)

---

## 중복 판정 기준

두 버그가 같은 패턴인지 판별하는 기준:

1. **같은 카테고리**에 속하고
2. **근본 원인이 동일**하거나 매우 유사하고
3. **예방 스펙이 동일한 해결책**을 제시

→ 위 3개 모두 충족 시 **중복** → 빈도수 증가 + 새 사례만 추가

→ 하나라도 다르면 **새 패턴** → 신규 항목 추가

---

## 예방 스펙 작성 규칙

예방 스펙은 PRD에 직접 삽입되므로 다음 규칙을 따른다:

1. **구체적**: "에러 핸들링 필요" (❌) → "네트워크 에러 시 '연결을 확인해주세요' 토스트 3초 표시 + 재시도 버튼" (✅)
2. **액션 가능**: 개발자가 바로 구현할 수 있는 수준
3. **클라이언트/서버 구분**: "클라이언트 사이드 파일 크기 검증 (10MB) + 서버 사이드 이중 검증"
4. **사용자 경험 포함**: 기술적 처리 + 사용자에게 보이는 결과

---

## 메타데이터 업데이트 규칙

매 학습 실행 시:
- `Last updated`: 현재 날짜로 갱신
- `Total patterns`: 전체 패턴 수 재카운팅
- `Sources`: 새 소스 추가 (기존 소스와 중복 시 카운트만 증가)

## 빈도 등급 기준

| 등급 | 횟수 | PRD 적용 우선순위 |
|------|------|-----------------|
| **높음** | 5회 이상 | 반드시 적용 |
| **중간** | 3-4회 | 적용 권장 |
| **낮음** | 1-2회 | 선택적 적용 |

`/generate-prdv2`에서는 **높음/중간** 패턴을 우선 적용하고, **낮음**은 관련 화면이 있을 때만 적용한다.
