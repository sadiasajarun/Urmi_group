# Service Launch Checklist

Master checklist for launching client services (App / Web / Both).
Each item has a **Condition** column — skip items whose conditions don't apply.

> **How to use**: At project kickoff, determine the project profile (app/web, free/paid, individual/organization, payment feature) and filter out non-applicable items. Items marked `ALL` are always required.

---

## Legend

| Tag | Meaning |
|-----|---------|
| `ALL` | Always required |
| `APP` | App (iOS/Android) only |
| `WEB` | Web service only |
| `ORG` | Organization enrollment (법인) |
| `PAID` | Paid app or subscription |
| `PAY` | Self-hosted payment feature (PG integration) |
| `iOS` | iOS only |
| `ANDROID` | Android only |
| `SOCIAL` | Has social login (Google, Kakao, etc.) |
| `HEALTH` | Health/medical app |
| `TRACK` | Uses tracking/advertising SDKs |

## Critical Path Timeline

```
Week 1 ──────────────────────────────────────────────── Week 4+
  │
  ├── 1-2. D-U-N-S application (ORG only) ────── 2-4 weeks ──────┐
  ├── 1-1. Apple Developer enrollment ──── after D-U-N-S ─── 1-3d ┤
  ├── 1-3~5. Google Play account + verification ──── 3-7 days ────┤
  ├── 4-1. PG contract (PAY only) ──── 1-2 weeks ────────────────┤
  ├── 2-1. Domain purchase ──── immediate ────────────────────────┤
  ├── 3-1~3. Legal documents ──── 3-7 days ───────────────────────┤
  ├── 5-1. App icon from designer ──── designer schedule ─────────┤
  │                                                                │
  └──────────── All complete ─────────────────────────────────────→ Submit
```

**Rule of thumb**: Start Section 1 (Accounts) and Section 5-1 (App icon) on project day 1.

## Quick Profile Filter

At project kickoff, answer these 5 questions to determine which tags apply:

| Question | → Tags |
|----------|--------|
| App, Web, or Both? | `APP` / `WEB` / both |
| iOS, Android, or Both? | `iOS` / `ANDROID` / both |
| Individual or Organization? | `ORG` if organization |
| Free or Paid? | `PAID` if paid/subscription |
| Self-hosted payment (PG)? | `PAY` if yes |
| Social login? | `SOCIAL` if yes |
| Health/medical domain? | `HEALTH` if yes |
| Tracking/ad SDKs? | `TRACK` if yes |

Then scan the checklist and **skip any row whose Condition you don't have**.
